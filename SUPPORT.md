# Support

Use the right channel so the project stays searchable and maintainable.

## Where to ask

- Usage questions and troubleshooting: open a GitHub issue with enough context to reproduce the problem.
- Bug reports: use the bug report template.
- Feature ideas or project direction questions: use the feature request template and explain the use case.
- Security issues: follow [SECURITY.md](SECURITY.md) and email siddharth.rathore2612@gmail.com. Do not file public issues.
- Code of conduct concerns: email siddharth.rathore2612@gmail.com and reference [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## What to include

- tracely360 version
- Python version
- Operating system
- Exact command you ran
- A small reproduction or sample files
- The relevant output or traceback

## Response expectations

- Support is best effort.
- Public threads are preferred so future users can find the answer.
- If there has been no response after 7 days, add a short follow-up to the same thread.

## Not supported

- Private debugging sessions for general usage questions
- Requests without reproduction details
- Security disclosures through public issues
